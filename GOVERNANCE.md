# Governance / 治理

## English

HermesX uses a lightweight maintainer-led governance model.

Maintainers are responsible for:

- Reviewing and merging pull requests.
- Keeping public documentation, release notes, and security guidance accurate.
- Protecting project quality, user safety, and license clarity.
- Making final calls when implementation tradeoffs conflict.

Contributors are encouraged to:

- Open focused issues or pull requests.
- Explain evidence, reasoning, and compatibility impact.
- Keep changes scoped and include verification steps.
- Follow the [Code of Conduct](CODE_OF_CONDUCT.md) and [Contributing Guide](CONTRIBUTING.md).

Decision policy:

- Small fixes may be merged after maintainer review.
- User-facing behavior, security posture, public APIs, release process, or governance changes should include rationale in the PR.
- Security-sensitive decisions follow [SECURITY.md](SECURITY.md) first.

## 中文

HermesX 采用轻量的维护者主导治理模型。

维护者负责：

- 评审并合并 pull request。
- 保持公开文档、发布说明和安全指引准确。
- 保护项目质量、用户安全和许可证清晰度。
- 在实现取舍冲突时做最终判断。

我们鼓励贡献者：

- 提交聚焦的 issue 或 pull request。
- 说明证据、推理和兼容性影响。
- 控制变更范围，并提供验证步骤。
- 遵循 [行为准则](CODE_OF_CONDUCT.md) 和 [贡献指南](CONTRIBUTING.md)。

决策规则：

- 小型修复经维护者评审后可合并。
- 影响用户行为、安全边界、公共 API、发布流程或治理方式的变更，应在 PR 中说明原因。
- 安全敏感决策优先遵循 [SECURITY.md](SECURITY.md)。
