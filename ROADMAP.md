# Roadmap / 路线图

## English

HermesX tracks released and unreleased capability status in the [README](README.md), [Changelog](docs/CHANGELOG.en.md), and [Enterprise Readiness](docs/ENTERPRISE_READINESS.en.md) matrix. This roadmap is intentionally lightweight and describes direction, not a release commitment.

Near-term priorities:

- Keep the current `v2.4.0-dev` documentation/API baseline aligned with the next stable release.
- Harden enterprise validation evidence for OIDC, sandbox modes, observability, and backup/restore.
- Keep public docs, examples, and GitHub templates accurate for new contributors.
- Continue improving governed agent execution, workflow automation, and multi-tenant operations.

Not currently in scope:

- A billing or invoicing product beyond usage records and aggregation.
- Public guarantees for unreleased features before they are cut into a stable release.
- Support for insecure-by-default deployment profiles.

## 中文

HermesX 在 [README](README.md)、[Changelog](docs/CHANGELOG.md) 和 [企业就绪度矩阵](docs/ENTERPRISE_READINESS.md) 中跟踪已发布和未发布能力状态。本路线图刻意保持轻量，只描述方向，不构成发布承诺。

近期重点：

- 保持当前 `v2.4.0-dev` 文档/API 基线与下一次稳定发布对齐。
- 补强 OIDC、沙箱模式、可观测性、备份恢复等企业验证证据。
- 保持公开文档、示例和 GitHub 模板对新贡献者准确可用。
- 持续改进受治理的 Agent 执行、工作流自动化和多租户运维能力。

当前不在范围内：

- 超出用量记录和聚合能力的 billing/invoicing 产品。
- 在稳定发布前对未发布能力做公开保证。
- 支持默认不安全的部署形态。
